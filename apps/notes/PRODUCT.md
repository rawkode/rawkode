# Product

## Register

product

## Users

People who use notes as an operating surface for daily work, research, planning, and structured personal knowledge. They want a local-first app that starts with a daily note, but can promote captured text into typed entities, relationships, and reusable views without switching tools.

## Product Purpose

Notes is a universal iOS and macOS notes app built around first-class daily notes, local SQLite persistence, a rich TipTap editor, typed supertags, entity relationships, and query-driven views. Success means a user can write naturally in a daily note, capture entities and properties in place, and inspect the resulting database through native views without the app becoming an outliner clone or a markdown-file toy.

## Brand Personality

Quiet, precise, durable. The interface should feel like a focused local tool for repeated work: dense enough for serious notes and databases, but still calm enough to live in all day.

## Anti-references

Do not make this feel like a marketing dashboard, a decorative card grid, a Markdown file browser, or a Tana-style outliner. Avoid oversized hero treatments, one-note color themes, novelty visuals, and UI copy that explains the app instead of helping the current workflow.

## Design Principles

- Daily notes are a primary workspace, not a shortcut to a document list.
- Structured data should emerge from writing, then become inspectable and editable through native surfaces.
- Relationships and backlinks should be visible where they affect the current note.
- Query views should feel like saved working views, not reports bolted onto an editor.
- Prefer clear, compact controls and platform conventions over custom decoration.

## Accessibility & Inclusion

Target WCAG AA contrast for custom surfaces, respect platform reduced-motion settings, keep controls keyboard and VoiceOver accessible, and avoid relying on color alone for note, entity, query, or relationship state.
