# Product

## Register

product

## Users

Enchiridion is for a single operator using notes as an active workspace: writing daily notes, capturing source material, tracking projects, and delegating recurring context work to agents. The user moves between writing, planning, triage, and review, often while pulling context from email, calendar, GitHub, bookmarks, and generated mini apps.

## Product Purpose

Enchiridion is a private second brain on Cloudflare. It combines daily notes, rich text editing, a resource graph, extension points, and Flue-powered agent workflows so new data stores and workflows can be added as mini apps when needed. Success means the user can write naturally, create structured app surfaces on demand, embed those app views in notes, and trust scheduled assistant work because every action is scoped and auditable.

## Brand Personality

Quiet, precise, extensible. The product should feel like a capable operator console for personal knowledge work: dense enough for repeated use, calm enough for writing, and explicit about what agents and mini apps are allowed to do.

## Anti-references

Avoid marketing-style heroes, decorative cards, novelty controls, vague AI chat widgets, and generated app surfaces that feel detached from the host workspace. Do not make mini apps arbitrary script injection into notes. Do not hide permissions, scheduled work, or external connector activity behind friendly but ambiguous labels.

## Design Principles

- Daily notes are the working surface, not a folder convention.
- Extensions start dynamic but must have clear contracts before they touch host data.
- Agent autonomy is acceptable only when capabilities, audit logs, and rollback paths are visible.
- Host primitives should render embedded work unless a mini app graduates into a native component.
- External systems become evidence in the resource graph, not disconnected feeds.

## Accessibility & Inclusion

Target WCAG AA contrast and keyboard-accessible controls. Support reduced motion. Do not rely on color alone for validation, destructive actions, permissions, or workflow state. Keep writing and command workflows usable from the keyboard.
